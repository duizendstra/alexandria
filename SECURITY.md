# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any Alexandria package, please
report it responsibly.

**Email**: jasper@duizendstra.com

Please include:
- The affected module and version
- Steps to reproduce
- Potential impact

I will acknowledge receipt within 48 hours and aim to release a fix within
7 days for critical vulnerabilities.

## Supported Versions

Only the latest minor version of each module receives security updates.

## Scope

This policy covers all artifacts published from this repository, including
Go modules under `go/`, API contracts under `contracts/`, and any generated
code or templates distributed through `blueprints/` or `skills/`.
